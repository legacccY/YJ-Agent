- **可接受的最坏结局**：______（如"至少能落一篇退路档 workshop"=底线）

## F. 默认 Kill Criteria（事前写死，G2-G5 机器执行）

> Gary Klein/Kill-criteria 原则：在情感依附形成前写下"满足什么条件就砍"。书面、不可中途随意松动。

| 闸 | 默认 kill 条件 |
|---|---|
| G2 撞车 | 与已发论文 SPECTER2 余弦 > **0.85** 且无差异化角度 → 砍 |
| G2 资源 | 估算算力 > C 节预算上限 → 砍 |
| G2 gap | 检索不到任何 future-work/limitation 支撑该问题是真 gap → 砍 |
| G3 可行性 | InnoEval Feasibility < **4**/10 → 砍（Hamming：no reasonable attack = not important）|
| G3 新颖 | InnoEval Novelty < **5**/10 → 砍 |
| G4 红队 | skeptic 给出 ≥1 个无出路的 🔴 致命 → 砍或回 G1 重挖 |
| G5 预实验 | <1 GPU·h 杀手锏证伪了核心 claim（如 baseline 已顶满 / 信号不存在）→ 砍 |

可在本轮按需收紧（如把撞车阈降到 0.80），但**只能更严不能更松**，松动需用户拍板并留痕。
