## 3. 三轮独立核查（本会话做了什么）

| 轮次 | 角色 | 结论 |
|---|---|---|
| **核口径** | verifier | paper Table 1 真源 = `itb_results.csv` 主管线收敛 ckpt；seed 那套量级差 4 倍纯因 ckpt 没训完；校准裁决在唯一有效口径下成立（F 不优于 D） |
| **修 bug** | coder | seed 复制 = 训练 bug（G=F 副本），非 eval bug。已拆 `run_g_tokft_seeds.py` + 加 F==G MD5 哨兵 + 6 pytest 绿 |
| **取证 0.707** | coder | 判定②幽灵数：旁支管线误抄进 abstract，原始不可考 |
| **判死活 R1-R5** | planner + coder | 零 GPU 离线判定，四卖点只一张 PASS（见下） |
| **审计 BMVC** | verifier | BMVC 命脉扎实，血统干净（见 §4） |

### R1-R5 判死活结果（零 GPU，本地离线重算）
目标 = 烧大卡重训前，先判「Q-VIB 不靠 AUC 赢的卖点到底立不立」：

| 卖点 | 结果 | F vs 自家最简 D |
|---|---|---|
| C-α 校准增量 | **FAIL** | ECE 差值 CI=[−0.062, +0.063] 含 0 = 无区别 |
| C-γ ρ 质量耦合 | **FAIL** | F Pearson −0.148 vs D +0.014，但 CI 重叠（Spearman 0.992 是 qbar 范围窄的假象） |
| **C-ε AURC / selective prediction（胜负手）** | **PASS** | F=0.201 < A(Direct)=0.348 ✅，但 **F vs D 不分（0.201 vs 0.210）** |
| C-β triage 反超 | **FAIL** | VIB 家族结构弱点：高熵把正样本全转介，sens_auto F=0.117 vs A=0.750 |

**致命点**：每一个 F vs D 对比都 FAIL。唯一活着的信号（VIB 在 selective prediction 上赢 Direct）是 **VIB 家族通性，跟 Q-VIB 这个具名贡献无关**——最简单的 Std VIB 一样能拿到。

**机理诊断**：VIB 全家在高质图上 AUC 也只有 0.59-0.61（bottleneck 把判别信息压到欠拟合），不是低质毁的。**AUC 赢 Direct = 数学性证伪**。R6 重训（放松 β/KL）最多把 F 拉到持平 Direct，**拉不出 Q-VIB 对 Std VIB 的差异化** → 判大概率白烧 20-30 GPU·h。

## 4. BMVC ↔ ICLR：同一套数据，为什么一个活一个死

**差别在「claim 了什么」，不在数据。**

| 维度 | BMVC（已投，活） | ICLR（启动中，死） |
|---|---|---|
| claim 类型 | **校准**：post-hoc QCTS 把 ECE 0.146→0.079 | **方法更强**：Q-VIB Full 赢 baseline 诊断 AUC |
| 「ours」是什么 | 朴素 **Std VIB + QCTS**（frozen 模型加 post-hoc 温度） | **Q-VIB Full**（复杂可训练 bottleneck） |
| 对 Direct 的态度 | **诚实承认** Direct AUC 最高、把它当前提（高准确但质量盲） | 想 claim 全面碾压 → 数据打脸 |
| 致命事实「Q-VIB Full≈Std VIB」 | **无伤**（从没说 Q-VIB 特别） | **致命**（具名贡献立不住） |
| 数据血统 | 收敛 ckpt、无 0.707 幽灵、QCTS 0.079 有真 csv（`qcts_itb_predictions.csv` 实算 0.0792） | 0.707 幽灵 + epoch=0 坏 seed ckpt（ICLR 后期代码漂移+multi-seed 独有） |

**BMVC 审计结论**：头条 QCTS ECE 0.079、D ECE 0.146、TS 反转 ρ+0.241 全有真 csv 撑，用的是收敛 ckpt。唯一缺口：zero-shot D+QCTS ρ（HAM −0.108 / PAD −0.150）的 `hamzero/padzero_predictions.csv` 不在磁盘（supp 次要数字，主表头条不受影响；若 BMVC rebuttal 需要再补）。

**关系本质**：这条线真正成立的成果（post-hoc 质量条件化校准）BMVC 已经拿走。ICLR 是把这个 post-hoc 修复**过度延伸**成「可训练 SOTA 系统」，而可训练版本打不过 baseline。

## 5. ICLR 命运：四条路 + 诚实推荐

> ⚠️ 死结：ICLR **不能重做 QCTS**（那是 BMVC 已投的贡献，重复 = 增量/自我抄袭）。

| 选项 | 内容 | 我的判断 |
|---|---|---|
| **① 调故事 + 降会场** | 砍掉「Q-VIB 更强」的 SOTA claim，改成诚实的 analysis paper：「降质医学图像上不确定性方法的 selective-prediction 可靠性（C-ε，唯一 PASS）+ quality-conditional IB 理论框架（5 定理已推完）」。投 TMLR / MICCAI / analysis track。 | 能立，最诚实，但要接受**不是顶会 SOTA、贡献从 Q-VIB 降级为 VIB 家族性质**，且要小心别跟 BMVC 重叠 |
| **② 暂搁 ICLR** | 冻结存档，精力转 NCA-JEPA（岔路口）/ MedAD-FailMap / 新方向。 | 省算力。鉴于救回来也只是中低会议 analysis paper，性价比要你权衡 |
| **③ 偏要试 R6 重训** | 烧 20-30 GPU·h 放松 bottleneck（β÷5 / KL-warmup）重训，赌能拉出 Q-VIB 差异化。 | **不建议**——问题是结构性的（Q-VIB≈Std VIB），重训改不了。可走卡槽调度试，但我判大概率白烧 |
| **④ 找全新角度** | 找 BMVC 不覆盖、且现有数据支持的新 claim。 | 难——数据已说 VIB 方法在本任务不差异化；要新角度可能要新数据/新任务（=接近新立项） |

**我的诚实推荐**：①或②二选一。BMVC 是这条线该守住的成果（已投、等录用）；ICLR 在硬撑一个数据不支持的更大 claim。若舍不得 ICLR 的 5 定理理论投入 → 走①（理论 paper + selective-prediction 实证，降会场）；若判性价比不够 → 走②暂搁，让位组合台其他战线。**不建议③烧卡。**
