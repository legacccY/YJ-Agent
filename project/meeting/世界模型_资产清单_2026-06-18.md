> **G5 rollout 可操作定义（关键设计决断）**：真纵向/时序医学数据本地全无，G5 用 BraTS 同 patient 相邻 slice 构成空间序列做最小 rollout（k=1,2,4,8）。诚实标注=slice 轴是空间 rollout 非疾病进展时间 rollout；只验「误差/方差是否随自回归步数复合」这一与序列轴正交的机制性问题，活过 G5 的首轮主实验必须换真纵向数据复验。

### 1.3 C047 去风险三轮迭代（最深一条，最终 INCONCLUSIVE 存档）

用户拍 B+C 组合：补 C047 真跨模态去风险。三轮迭代全程双红队连环拦：

1. **真跨模态版（AbdomenMRCT，非 CHAOS）**：CHAOS 实为不配对，改用 Learn2Reg AbdomenMRCT（真配对同解剖 + 器官分割）。两道健全性闸 3/3 seed VALID（真改图 + 跨模态 D_pre 真大），resolve_ratio≪0.5 字面 PASS。但 **analyst 终判=条件 PASS（不够干净）**：seg_mask rr≈0 分不清「VAE 结构不变性」vs「2000 步欠训编码器忽略输入」；latent rr 负是 **OOD artifact**（对齐后图对 MRI 编码器 OOD）非「dynamics 顽固」。根本混淆=「对齐外观」与「喂 OOD 图」在此设计等价。

2. **pixel-space 补偿实验设计 → skeptic 红队 3 致命（未跑）**：
   - 致命-1/2（纯数学）：pixel 度量配直方图对齐 → D_post 被对齐机械压到 0 → rr→1 趋 KILL，换成「度量-对齐共线」新混淆，反向自动误杀 claim，比首版更危险。
   - 致命-3（构念效度，深层）：首版+本版均为**两个独立单模态 WM**（CT/MRI 各跑各不交互），测的是「两独立模型预测差」（本就该不同=平凡），根本没测「跨模态学习时 dynamics collapse」。C047 真 novelty 需**共享/联合跨模态 WM**。当前=弱命题「单模态 WM 不可迁移」，撑不起 ICLR。

3. **强命题 shared cross-modal WM（RUN-2d）→ INCONCLUSIVE 存档**：
   - 用户拍强命题 → 重设单一 shared WM（vae_shared+dyn_shared，CT+MRI 混训）+ 纯结构度量 S1 Dice 残差/S2 质心/S3 Hu 矩（与强度对齐数学正交）+ 三 null 基线 + 闸 C 改 AND。skeptic 二次复审 0 致命。coder 47 pytest 过。
   - HPC 跑 4 轮（1461557→1462535）+ 三轮地基诊断：**核心死因=VAE recon 质量不足**，latent32+shared WM **mode-collapse**（2/3 seed 系统性 collapse），recon 近均匀无结构 → Otsu/p75 退化输出空 mask，nullA_floor S1 残差始终 0.5~0.6（spec 预期≈0）。
   - **增训练步也救不活（决定性 diag job 1462535）**：seed0 2000 vs 10000 步两步数 gate_C_var 全 = 0.00e+00（72/72 行 INVALID_MODE_COLLAPSE），loss 仅 0.0657→0.0584。**5 倍训练 mode-collapse 纹丝不动 = 容量/稳定性问题非欠训练**。
   - **闸 C 价值兑现**：skeptic 加的 decode-非退化 AND 闸正确拦下全部假信号（mode-collapse recon 不放行假 PASS）。
   - **终判 INCONCLUSIVE**：G5 toy-proxy（latent32 ConvVAE+GRU+16 配对）不具备可靠结构分析条件，撑不起 C047 强命题去风险。
