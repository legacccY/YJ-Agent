# NCA-JEPA 创新计划（方法名 SCP-JEPA）

**稳定细胞预测器驱动的联合嵌入世界模型**
从「NCA 替代 ViT」转向「NCA 解锁 ViT 做不到的事」

> **命名规范（单一真源）**：
> - **NCA-JEPA** = 项目 / 架构家族名（本文件夹、arXiv 占坑名，2026-06 实测搜索未被占用）
> - **SCP-JEPA**（Stable Cellular Predictor JEPA）= 加了稳定化三件套的具体方法 = 实验 A2 臂
> - 文中「我们的方法 / 主臂」一律指 SCP-JEPA；「这条研究线 / 框架」一律指 NCA-JEPA

## 一、一句话和核心叙事

**一句话**：把医学影像世界模型（JEPA）里那个又重又黑盒的 Transformer 预测器，换成一个会"自己生长出答案"的神经细胞自动机（NCA）。我们不比谁分数高，而是证明这个细胞预测器能做三件 Transformer 做不到的事——训练稳定性可分析可控、推理随时可停、生长过程可看。

**为什么这么定位**：直接打"NCA 比 ViT 好"这一仗大概率打不赢，理由有三：

1. **省参不是真痛点**：JEPA 预训练的成本大头在 ViT 编码器（86M 参数），预测器省 100 倍参数对总成本几乎没影响。
2. **等变优势不牢靠**：ViT 配足够的旋转数据增强能学出近似等变；CheXWorld 的域变化建模也在做类似的事。
3. **稳定性是动力学本质问题**：复现报告显示前列腺 0/11 训满轮数、同种子落进不同盆地。光靠 spectral norm 降低放大因子，但长链迭代的指数敏感性还在——这本身就是个值得研究的问题，不是缺陷。

所以换打法：**不证"更好"，证"独有"**。

## 二、背景：两篇论文在讲什么

### Med-NCA（Kalkhof 2025，医学影像顶刊 MedIA）

把分割想象成**一群细胞各自跟邻居反复聊天，慢慢长出器官轮廓**。NCA = 一个很小的网络（1–7 万参数，树莓派能跑），套到每个像素上重复跑固定步数，全局结构靠局部通讯涌现。每步有随机激活（fire-rate），同图跑十次结果略不同，取平均更稳，方差还能当"模型有没有把握"的信号（NQM）。

**它的命门**（我们复现亲手撞到）：高分辨率任务训练会发散，11 次没一次训到论文轮数；推理步数必须等于训练步数。

### CheXWorld（Yue 2025，CVPR）

把自监督想象成**让模型当放射科医生，看到胸片一部分就脑补出看不见的部分**。三个任务：遮一块补一块（局部结构）、给两块相对位置猜另一块（全局布局）、换曝光看穿成像差异（域变化）。关键部件是一个 **ViT 预测器**：输入"看得见的内容 + 一个条件 z"（遮罩/相对位置/增强参数，相当于世界模型里的"动作"），输出"看不见那块的特征"。8 个 benchmark 刷到 SOTA，代价是 ViT-Base + 8 卡 16 小时。

### 同构关系

两篇骨子里做同一件事：**给定上下文 + 一个动作 z，预测目标状态**。CheXWorld 用 ViT 一次算出来；NCA 用小网络一轮轮长出来。**NCA 本身就是一个空间局部、权重共享的世界模型**——把它放进 JEPA 框架，就是这个方案。

## 四、NCA 的三张牌（按抢占后的真实强度重排）

| 能力 | NCA 预测器 | ViT 预测器 | 强度（抢占后） |
|---|---|---|---|
| **① 训练稳定性可分析可控 + 机制表征** | 谱归一约束膨胀 → latent 趋准不动点（经验，非全局收缩）；我们有独家发散数据解释为何旧做法崩 | 单次前向，无此问题也无此分析 | 🟢 **主打**。用独家数据，是真贡献 |
| **② 随时推理（anytime）** | 迭代天然可提前终止，无需额外训练目标 | 默认全或无；加早退头（A0+）可 anytime 但需额外 N 个 head + 等权 loss，且 SSL predictor 位无先例 | 🟢 **主打**。卖点精确为「架构原生 anytime + 稳定区仍保质」，由 A0+ 对照 + §9.1 trade-off 坐实，非空说「ViT 做不到」 |
| **③ 可解释生长轨迹** | 每步通道状态可视化，看预测如何长出来 | ❌ 注意力图粗粒度 | 🟡 次要（定性加分） |
| **④ 内建不确定性** | fire-rate 方差 → 质量信号 | ❌ 需外挂贝叶斯头 | 🟡 次要，**且被 resilience 抢**，须对标区分 |

**moat 逻辑**：即使下游分数略低于 ViT（≤3 AUROC），①② 这两项是 ViT 预测器结构上给不出的，且 ① 有我们独家数据撑。这构成"capability paper"而非"SOTA paper"的护城河。

## 五、方法框架：SCP-JEPA

### 5.1 整体结构

与 CheXWorld 共享编码器（pilot 用 ViT-S，主实验 ViT-B），**只换预测器**：

```
Input → Context Encoder (ViT, EMA→Target Encoder) → NCA Predictor → 预测的目标特征
                                                          ↑
                                                   条件 z (mask / Δ / a)
```

NCA 预测器内部（在 latent token 网格上，不是像素）：

```
输入：看得见的 context 特征 h_x + 带位置编码的待填 mask token
重复 S 步：
  感知：Conv3×3(h) + identity   → 局部邻域
  更新：fc₁(ReLU(fc₀(感知)))     → 残差增量
  随机激活：Bernoulli(0.5)        → fire mask
  残差：h ← h + fire_mask ⊙ 更新
  锚定：h[0] ← 输入特征            → 防漂移
输出：预测的目标特征 ĥ_y
```

> 注：原版 Med-NCA 的"128 步链"是像素空间两级 ×64 步的设定。在 latent 14×14 网格上做单级预测器，步数 S 会小得多，指数敏感性也弱得多——但仍需稳定化，见下。

### 5.2 稳定化三件套

**问题本质**：NCA 预测器是一个迭代动力系统 h_{t+1} = h_t + m_t ⊙ f_θ(h_t)。某步 Jacobian 谱半径超过 1，扰动就指数放大；长链迭代下，连 ~10⁻⁷ 的 cuDNN 浮点扰动都能在终点放大成显著差异（复现报告实测）。

| 组件 | 机制 | 理论依据 | 实现 |
|---|---|---|---|
| Spectral Normalization | 约束每步更新的 Lipschitz 常数 < 1 | 压缩映射 + Banach 不动点：保证迭代收敛到唯一不动点 | `spectral_norm` 包 conv/fc 层 |
| EMA Target Anchoring | EMA 目标编码器给稳定预测目标，防止预测器追自己的漂移 | I-JEPA 原版机制，此处额外起稳定锚作用 | 复用 I-JEPA EMA，momentum 0.996→1.0 |
| Deterministic Mode | 关 cuDNN benchmark、开 deterministic algorithms、固定 fire mask 种子 | 消除 cuDNN/atomic 不确定性，使种子级复现可能（复现报告 §4 直接教训） | `cudnn.deterministic=True`、`benchmark=False`、`use_deterministic_algorithms(True)` |

### 5.3 三个能力的形式化（一处讲清，不再重复）

- **① 稳定性**：Lipschitz<1 → rollout 收敛到唯一不动点 h*(z)，对 fire-rate 随机性敏感度有界。配独家失败数据论证"像素迭代踩在边界、latent+收缩拉进稳定区"。
- **② 随时推理**：质量曲线 Q(k) = 第 k 步提前终止的预测 vs 满步预测的 cosine 相似度。算 Q(1,2,4,8,16,32,64)，看是否前期快速收敛。
- **③ 可解释**：记录每步通道状态，NCA-VIS 风格可视化"从模糊全局猜测 → 逐步细化局部解剖"的过程。
- **④ 不确定性（次要）**：跑 K 次（fire mask 不同），σ_y = std(预测)，看 ‖σ_y‖₂ 与预测误差的 Spearman ρ。**实验须同台对标 resilience（扰动推理态）和 MC dropout**，讲清差异（训练原生 vs 外挂）。

## 六、第一步：48 GPU 小时可行性预检

**最重要的模块**。投入 4 周主实验前，用极小规模验证 4 个最危险假设。任一红灯先修或转路线，不硬推。

| # | 假设 | 检法（ViT-S，NIH 10K 子集，mask-only JEPA） | 时间 | 绿灯标准 |
|---|---|---|---|---|
| PC-1 | NCA 训纯预测任务（latent L2）比训分割（Dice）稳 | 无约束 NCA 预测器，3 seed × 50 ep | 6h | 3/3 无发散、塌缩 canary 健康。0/3 存活 → NCA 连纯预测都稳不住，直接转路线 B |
| PC-2 | 稳定化三件套压得住塌缩 | 同上加三件套，对比 PC-1 | 6h | 3/3 存活 + loss 平滑 + 终点 loss 跨 seed std<0.1。仍不稳 → 加梯度裁剪+降 lr 再一轮 |
| PC-3 | fire-rate 方差编码预测质量（不确定性牌） | PC-2 冻结 ckpt，10× 推理，方差 vs 预测误差 Spearman ρ | 2h | ρ≥0.3, p<0.05。ρ≈0 → 不确定性牌彻底放弃（本就被抢，不强求），叙事重心压到 ①②③ |
| PC-4 | NCA 预测器下游不掉崖 | PC-2 ckpt，VinDr-CXR 1% linear probe，对比 ViT 预测器 | 8h | ΔAUROC>-5。差>5 → 转路线 C |

**总计 ~48 GPU·小时（单卡 4090），2 天内完成。**

预检后分路：
- 全绿 → 进 Phase 1
- PC-1/PC-2 红 → 路线 B（稳定性理论论文）
- PC-1/2 绿、PC-4 红 → 路线 C（诊断工具叙事）
- PC-3 红 → 不确定性牌弃用（本来就弱），主推 ①②③

## 七、对照臂（4 臂，控变量）

| 臂 | 预测器 | 步数 | 验什么 |
|---|---|---|---|
| A0 | ViT 预测器（I-JEPA 原版，单次前向） | 1 | 管线 sanity + 下游上限 |
| **A0+** | **early-exit ViT 预测器**（每层接 L2 head，等权聚合，MSDNet 式） | 1…N（可早退） | **②牌正面对手**：努力过的 ViT 也能 anytime，证 NCA 逐步 anytime 是否真更优（详见 `03_pilot §5`） |
| A1 | vanilla NCA（无约束） | S | 反例：直接搬入是否仍塌缩（故事需要） |
| A2 | 稳定 NCA（三件套） | S | 主臂：稳定化是否有效 + 下游是否不劣于 A0 |

- 全 NCA 臂预测器参数 ≤ A0；A0+ 只多 N 个线性头 ≈ A0（不偷加参，与 NCA 臂同量级，公平对比）。
- A1 时间紧可砍，但留着能讲"问题→修复"。
- **A0+ 必留**：reviewer 认定不补 early-exit ViT 对照，②牌会被「ViT 也能 early-exit」一句打穿（reject 级）。

## 八、量化指标与决策门

### Pillar 1 — 稳定性

| 指标 | 绿灯 |
|---|---|
| Loss 轨迹 | 无 NaN、无 flat-high（>3 持续>5 ep）；3/3 seed 健康 |
| 塌缩 canary | token 嵌入 std / 有效秩 / KoLeo 熵全程高于阈值（取 A0 正常训练 50% 分位） |
| 跨 seed 一致性 | 3 seed 终点 loss std < 0.1 |
| 盆地双峰 | 无双峰，所有 seed ep1 loss 在同一簇（复现报告 §3.3 signature） |

### Pillar 2 — 表示质量（非劣效，不是要赢）

| 指标 | 绿灯 |
|---|---|
| Linear probe AUROC | VinDr/RSNA/ChestX-ray14/CheXpert @1%/10%/100%，A2 均值 − A0 均值 > −3 |
| 旋转 OOD | A2 掉幅 ≤ A0（副指标，不设硬门） |

### Pillar 3 — 独有能力

| 指标 | 绿灯 |
|---|---|
| **② 随时推理 × 稳定性 trade-off（一等指标）** | Q(k) 曲线族（多 L_f + A0+ 同台）+ anytime-gain vs stability-margin 散点；某 L_f≤0.9 稳定配置下 anytime-gain=Q(8)/Q(64)≥0.85 → 「稳定×anytime 可共存」成立。详见 `03_pilot §9.1`（阈值为工程 go/no-go，非论文 claim） |
| ③ 可解释 | 通道状态呈可解释解剖结构，≥2 人认可（定性） |
| ④ 不确定性（次要） | ρ≥0.3，且对标 resilience/MC dropout 有差异化说明 |

### 决策门（预定义 if-then，不达标按预案走）

- **Gate 0（预检，D2）**：PC-1~4 全绿 → Phase 1；PC-1/2 红 → 路线 B；PC-4 红 → 路线 C；PC-3 红 → 弃不确定性牌。
- **Gate 1（稳定性，D11）**：A2 3/3 无发散 + canary 健康 + 跨 seed std<0.1 + 无双峰 → 进 Gate 2；否则 → 路线 B（"我们发现 NCA-JEPA 的根本稳定性上限"本身是贡献）。
- **Gate 2（非劣效，D16）**：A2 vs A0 ΔAUROC>−3 → 进 Gate 3；差 3–5 → 放宽到 −5 并强化 ①②叙事；差>5 → 路线 C。
- **Gate 3（能力，D21）**：①②③④ 至少两项达标 → GREEN 完整推进；仅一项 → 以那项为核心；零项 → 路线 B。

## 九、Silent-bug 哨兵（强制，跑任何实验前先过）

继承 pilot + 复现报告教训。任一红灯 = 管线 bug，先修再跑。

1. **归一化 assert**：log 喂入张量 mean/std == 期望 NORM；raw 值溜进来直接 raise（上个项目 visiscore 喂 raw 的同类坑）
2. **EMA sanity**：target ≠ context；target encoder 不吃梯度
3. **塌缩 canary**：嵌入 std < 阈值 → flag（别把低 loss 当成功）
4. **overfit-one-batch**：全跑前单 batch loss 压到≈0；压不下 = 代码 bug，立即 abort
5. **z-shuffle 对照**：打乱条件 z → 下游应掉；不掉 = z 没接进去
6. **determinism 留痕**：记 cudnn flags + seed
7. **静默发散检测**：ep10 后 loss>3 持续 → 自动标 dead 取消（复现报告 §B.3）
8. **纯预测 overfit**：单图反复预测，10 ep 内 L2<0.01；压不下 = NCA 连预测都学不动，查架构

| **resilience（arXiv 2605.26726, 2026-05）** | **不确定性方向的竞品/抢占者**。它扰动推理态，我们是训练原生方差——实验须正面对标区分，叙事不依赖此牌 |
| MC Dropout（Gal 2016） | 不确定性 baseline，NCA 随机激活类似内置 MC dropout，但架构原生 |
| **Early-exit / anytime ViT（MSDNet ICLR'18 1703.09844；A-ViT CVPR'22 2112.07658；MeViT 2106.15183）** | ②牌的 ViT 侧对手，A0+ 臂采 MSDNet 等权聚合配方。关键：这些全在分类/判别 fine-tune 做 anytime，**无一在 SSL latent-regression predictor 位做早退** → anytime-SSL-predictor 仍空白，强化②牌 novelty |
| **迭代收敛-稳定 trade-off（Bassily 2018 1804.01619；DEQ Bai 2019 1909.01377）** | §9.1 一等指标的理论根：「收敛快⇄稳定差」+ 压缩映射 L<1 收敛快但表达受限，刻画 SCP-JEPA 稳定×anytime 张力 |
| Equivariant SSL（Dangovski ICLR 2022） | NCA 等变是涌现的，不需显式等变损失（但本方案不主打等变） |

**独特位置**：首次把迭代细胞预测器引入 JEPA，不追 SOTA，展示稳定性修复 + 随时推理 + 可解释这三项 ViT 预测器给不出的能力。

## 十五、审稿人预对峙（精简）

| 攻击 | 回应 |
|---|---|
| "只是换了预测器，novelty 不够" | 贡献不是换零件，是 (1) 首次表征并修复迭代预测器在 JEPA 中的稳定性，(2) 展示 ViT 预测器没有的随时推理 + 可解释 + 不确定性。这些来自架构本质，不是调参能得 |
| **"ViT 也能 early-exit / MSDNet 做 anytime，②牌打稻草人"** | 我们正面加 A0+（early-exit ViT predictor，MSDNet 等权聚合官方配方）同台对照，不打 vanilla 稻草人。两点回应：(1) 全网零 anytime-SSL-predictor 先例——early-exit 全在分类/判别 fine-tune，没人在 JEPA latent-regression predictor 位做过，ViT 早退在此场景同样是我们自建的；(2) NCA anytime 架构原生、无需额外训练目标，ViT 早退要多挂 N 个 head + 等权 loss。胜负由 §9.1 trade-off 实测，赢则报赢，平则诚实报「ViT 早退同样够用」 |
| **"稳定区=anytime 最弱区，自相矛盾"** | 承认这是真张力（Bassily 2018：收敛快⇄稳定差）。正因如此把 stability-vs-anytime trade-off 升为一等指标（§9.1），测「稳定×anytime 同时成立」的区间存不存在。若不存在，诚实报 best-compromise S，不藏 |
| "下游没超过 ViT" | 本工作是 capability paper 不是 SOTA paper：在性能不显著降低前提下提供额外能力 |
| "稳定性方案就是 spectral norm + 确定性，太简单" | 简单方案解决一个此前被忽视的系统性问题正是好论文特征；我们给了动力系统角度的理论分析说明为何必要 |
| "不确定性不就是 MC dropout / resilience 吗" | 与 MC dropout 区别：训练原生而非外挂。与 resilience 区别：它扰动推理收敛态，我们用训练原生 fire-rate 方差；实验同台对标。且本方案不依赖此牌 |
| "为什么不用 NCA 替代 encoder" | NCA 迭代延迟高、不适合大批量预训练做 encoder；在预测器位置其局部性+随机性恰好转化为独有能力。encoder 的 ViT 贡献表示质量，predictor 的 NCA 贡献新能力，互补 |
