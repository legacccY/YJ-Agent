## 超参（researcher 2026-06-24 已清，官方源带引用）

### BrainGB（`HennyJie/BrainGB` argparse 默认）
lr=1e-4 / weight_decay=1e-4 / epochs=100 / hidden_dim=360(GCN) / n_GNN_layers=2 / n_MLP_layers=1 / dropout=0.5 / batch=16 / Adam / seed=112078。
- node feature = `adj`（`Adj()` transform，每节点特征=FC 行向量 connection profile）
- GCN mp=`weighted_sum`；GAT mp=`attention_weighted`，num_heads=2；pooling=`concat`
- split = `StratifiedKFold(n_splits=5, shuffle=True)`（代码默认 5，部分论文用 10——以代码为准）
- 报告 acc：GCN 0.644±0.038 / GAT 0.679±0.039（10-fold，BrainIB 引）；CC200 65.90%（mGNN-bw 引）；README 示例 65.31±1.58 → **对齐 65-70% 判据 ✅**
- TODO 残留：TMI 论文原始 Table 数字 PDF 提不出（第三方引用代替，足够做 Gate1 基准）

### HyperGALE（`mehular0ra/HyperGALE` 官方 yaml）
**数据=ABIDE-II + Schaefer400，n=812（ASD384/TD428），16 sites**（⚠️ 非 ABIDE-I）。
lr=1e-5 / base_lr=1e-4 / target_lr=1e-5 / weight_decay=1e-4 / poly scheduler power=2 / epochs=200 / hidden=64 / num_layers=1 / dropout=0.5 / batch=16 / Adam / repeat=10。
- 超边=k-NN，**K_neigs=40**（论文最优）；关联矩阵 H∈R^{N×K} + 可学习 W̃∈R^{K×K}；num_edges=node_sz=400
- attention hidden=256 / attention dropout=0.25 / LeakyReLU（hardcoded）；readout=linear
- node feat：Schaefer400 时序→Ledoit-Wolf shrinkage full correlation（nilearn）→400 维 FC 行
- split=LOSO（16 sites）+ 5 比例 90/10
- 报告 acc=**75.34±0.47%** / AUC=77.03±1.85
- ⚠️ **2 open issue（#2/#3）= `fc_large_data.npy` 预处理脚本缺失**：repo 不提供从原始 ABIDE-II 生成 FC 的脚本，**须自写 nilearn+Schaefer400+Ledoit-Wolf 管道**。不阻断模型码 + 不阻断 XAI 接管道。
- 环境：py3.11 / torch 2.0.1 / **PyG 2.3.1** / Lightning 2.0.7 / Hydra+wandb

### PyG fidelity / GNNExplainer
- `fidelity(explainer, explanation) -> (fid+, fid-)`；fid+ = 删解释子图看预测崩（补集 mask），fid- = 只留子图看是否够。
- `Explainer(model, algorithm=GNNExplainer(epochs=200), explanation_type='model', node_mask_type='attributes', edge_mask_type='object', model_config=dict(mode='binary_classification', task_level='graph', return_type='log_probs'))`
- GNNExplainer 默认 epochs=100/lr=0.01；官方 example 用 epochs=200
- nan 隐患：mask 全 0/全 1 可能 nan（主函数无显式检查）；regression 模式直接报错
- PyG latest=2.8，explain 模块 2.3+ 引入（与 HyperGALE 的 2.3.1 兼容✅）
- 🔴 **命门坐实**：PyG **原生不支持超图 fidelity**（issue #5630 Open/P0）。`fidelity()` 走 `edge_index`，HyperGALE 用 `hyperedge_index`（incidence），**无匹配 mask 接口** → run-06 直接套必报错/语义错。**必须自研 hyperedge mask 适配层** = 正是本文 Gate2 L1 协议贡献点，非 Gate1 阻碍。备选库 DeepHypergraph(DHG) 有超图 GNN 无 fidelity 接口。

**工程坎：**
4. PyG 不原生支超图——run-06 fidelity 套超图大概率 nan/语义错（预期探路，归 Gate2 非 Gate1 FAIL）。
5. node feature 定义 BrainGB vs HyperGALE 不同，各按官方不跨用。
6. atlas：HyperGALE Schaefer400 / BrainGB cc200——Gate1 各按官方 atlas，不强求同 atlas（同 atlas 可比=Gate2 L1 的事）。
7. split 以 SUB_ID 为键 + site-stratified 防 site 泄漏。
8. `.portfolio/datasets.json` 当前**无 abide 条目**，P0b 必补登记。
9. P1 HPC 上传=拍板点先报。

## Gate1 判定逻辑

**PASS（三必需全绿进 Gate2）：**
- ✅ P 阶段：三 atlas 下载 + 连接矩阵 + patient-level split 就位
- ✅ run-01 BrainGB GCN：ASD acc 65-70%（3 seed 均值），非发散
- ✅ run-03 HyperGALE：acc 对齐论文值，非发散
- ✅ run-04 PyG fidelity on BrainGB：fid+/fid- 非 nan 有限数（verifier Bash 核 csv）

**触发 K1 砍/停报：**
- ❌ ABIDE 下载不到 → K1
- ❌ baseline 反复发散 / acc 显著低于文献（<55% 近随机）且复现零偏离后仍不达标 → 停报候选 K1
- ❌ run-04 普通 GNN 都算不出非 nan → 工程坎升级停报
- ⚠️ run-06 超图 fidelity nan **不触发 K1**（Gate2 命门探路）；acc 区间 65-70% 不自创放宽
