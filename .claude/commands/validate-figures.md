# validate-figures

图表双重验证流程。每次生成论文图表后运行。

## 触发场景

用户运行了 `gen_*_figures.py` 或 `gen_figure_*.py` 之后，或说"检查图表"、"图做好了"。

## 执行步骤

### Step 1：读取图表脚本和输出文件

读取本次生成图表的脚本（gen_*.py），了解：
- 使用了哪些数据文件（CSV/JSON）
- 生成了哪些输出文件（fig_N.pdf / fig_N.png）
- 每张图的 X/Y 轴含义

### Step 2：自动一致性检查（读数据文件）

对每个输入数据文件执行：
1. **范围检查**：数据的实际 min/max 值是多少？
2. **数量检查**：数据点数量是否合理（> 5）？
3. **分组检查**：如果数据有分组（子集、类别），整体趋势是否可能被分组结构扭曲？

如果发现潜在的 Simpson's Paradox 风险（数据有自然分组，且分析了整体相关性），输出警告：

> ⚠️ 警告：检测到分组数据的整体相关性分析。请确认：
> - 整体趋势是否与各组内部趋势一致？
> - 分组是否是潜在混淆变量（例如图像质量层级、数据集来源）？
> - 如不一致，应优先报告分层结果，而非整体结果。

### Step 3：人工核查 Checklist 输出

为每张图输出以下 checklist，用户逐项确认：

```
Figure {N} — {图标题} 核查清单：

数据完整性：
[ ] 所有数据点在可视范围内（无截断）
[ ] X/Y 轴范围与数据实际 min/max 匹配
[ ] 数据点数量与预期一致（应有 N 条曲线/N 个点）

标注与标签：
[ ] X 轴标签与数据含义一致
[ ] Y 轴标签与数据含义一致
[ ] 图例覆盖所有曲线/分组，无遗漏

数字一致性：
[ ] 图中标注的数字（如 ρ = -0.192）与论文正文对应数字一致
[ ] 图中趋势与 Table 中对应行的数字方向一致

视觉质量：
[ ] 分辨率 >= 300 DPI
[ ] 字体大小在 A4 印刷后可读（建议 >= 7pt）
[ ] 颜色区分度足够（考虑色盲友好）

Simpson's Paradox 检查（若含相关性分析）：
[ ] 已考虑分层效应（分组内部趋势与整体趋势一致）
[ ] 使用了合适的数据来源（非人工分层的自然分布数据集）
```

### Step 4：通过标记

所有 checklist 项用户确认通过后：
- 输出：`✅ Figure {N} 验证通过`
- 创建标记文件：在图表所在目录写入 `fig_{N}.verified`（内含验证日期和 checklist 摘要）

### Step 5：汇总报告

所有图表检查完成后，输出：
```
图表验证汇总：
✅ fig_1: 通过
✅ fig_2: 通过
⚠️ fig_3: 待确认（checklist 未全部完成）
❌ fig_4: 发现轴域截断问题，需修复

下一步：修复 fig_4 的 X 轴范围，重新生成后再次运行 /validate-figures
```

## 常见问题参考

**轴域截断**：如果 MC Dropout/Deep Ensemble 等 baseline 预测集中在某个范围，不要把轴域设到只显示主方法的范围。用 `ax.set_xlim([data_min - margin, data_max + margin])`。

**Simpson's Paradox**：VisiSkin 项目中，用 ITB（按质量分层选出）的数据计算 entropy-quality 相关，会产生虚假的强相关。应改用自然分布的数据集（HAM10000 zero-shot）。
